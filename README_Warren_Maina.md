# 1. What is the role of a class in a linked list?

A class is as a blueprint for creating objects. In a linked list, classes are used to define nodes and the linked list itself. The Node class stores data and a reference to the next node, while the linked list class manages the nodes.

# 2. What is the difference between a node and a linked list?

A node is a single element that contains data and a pointer to the next node while a linked list is a collection of connected nodes that form a sequence.

# 3. Why do we use None in next?

We use None to show that a node is not connected to another node yet or that it is the last node in the linked list.

# 4. How is a linked list different from a Python list?

A Python list stores elements in contigous memory locations and allows direct access using indexes while a linked list stores elements as connected nodes, and elements are accessed sequentially.

# 5. Why is OOP useful for data structures?

OOP helps organize code using classes and objects. It makes data structures reusable, easier to manage and easier to understand.
